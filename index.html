Lesson 1: HTML Markup

What is HTML and its Purpose

HTML is an acronym for Hypertext Markup Language. Programmer's write code as HTML documents. Basically code tells the web page what it is going to look like.

Computers have no idea what you want them to do unless you write programming code that tells it exactly what you want the web page to look like in the end result. The next, two sections will give a basic understanding of HTML vocabulary.

Tags and Elements

The HTML Markup consists of tags and elements. Tags are contained within angled brackets. On the keyboard they are the less than and the greater than symbols. During the first lesson, the instructor used the Udacity playground for illustration.

The opening tag, the contents within and the closing tag comprise the code element.

Attributes and the HREF

Attributes are values that provides additional information, or configure the behaviour of a HTML elements in a way, which is appropriate for the end users. Some example of attributes can be, class, id, href, src, rel, and media. Whenever you look at a web site and it has images those images are attributes. The programming code for image is img src.

Sometimes when you read an article on the web it contains hyperlinks with references to other online articles. These attributes use a programming attribute name called href.

Lesson 2: HTML Documents

HTML Head and Body in Code Structure

When writing HTML code the programmer has to choose a software to write the code. Scratchpad.io is a basic HTML editor for beginners. In order to write the code the programmer has to give a title to the item and classify what it is.

The body of HTML is content. So the writing we read in paragraphs would be considered body in HTML. Content is the contents of the document.

HTML, CSS and Javascript

There are three, major dimensions to creating a web page. The HTML document is the structure of the web page. Without an HTML foundation there is no web page.

This structure consists of boxes that the programmer has to visualize when creating a web page. By looking at sections of a web page the programmer has to count the boxes so s/he can classify these boxes in her coding.

The next dimension of a web page is Cascading Style Sheets, also known as CSS. Web pages have color, backgrounds, borders, fonts...this is the style of the web page. CSS is written in a separate column from HTML and it tells the computer what artistic features to put on the page.

Another aspect of a web page is Javascript. Interactive components of a web page are a result of Javascript. Web developers usually assign the changing elements of a web page, the Javascript, to one programmer. More in-depth coverage of Javascript will be in future segments.

DOM Tree-Like Structure

According to Wikipedia, DOM "The Document Object Model (DOM) is a cross-platform and language-independent convention for representing and interacting with objects in HTML, XHTML, and XML documents.[1] The nodes of every document are organized in a tree structure, called the DOM tree. Objects in the DOM tree may be addressed and manipulated by using methods on the objects. The public interface of a DOM is specified in its application programming interface (API)."

DOM is the elements of a tree. HTML uses tags and these tags can be organized into a tree-like structure, which is DOM. DOM is an absolute component in a programmer's organization of the HTML documents they create.

CSS Inheritance

According to Wikipedia, "Inheritance is a key feature in CSS, it relies on the ancestor-descendant relationship to operate. Inheritance is the mechanismby which properties are applied not onlyto specified element, but also to its descendants. Inheritance relies on the document tree, which is the hierarchy of (X)HTML elements in a page based on nesting."

Programmers use this web site as their CSS reference.

Embed CSS Stylesheet into HTML

Within the HTML document, add a "head" tag at the top of your HTML. Then add a "link" tag. After "link" add the following: rel="stylesheet" type="css" and then href="main.css."

The Box Model

In the box model, the center is content. Surrounding content is padding, which is surrounded by border and then the border is encapsulated by margin. In CSS the programmer has to determine box width, height, margin, border-width, and border-color. Sizes can be determined in either percents or pixels.

HTML can be validated at: http://validator.w3.org/#validate_by_input

DRY: Do Not Repeat Yourself

As criteria for passing this phase of the project, the instructors want explanations as to why there should not be repetition in coding.

According to Wikipedia, DRY, or Do Not Repeat Yourself "In software engineering, donâ€™t repeat yourself (DRY) is a principle of software development, aimed at reducing repetition of information of all kinds, especially useful in multi-tier architectures. The DRY principle is stated as 'Every piece of knowledge must have a single, unambiguous, authoritative representation within a system.' The principle has been formulated by Andy Hunt and Dave Thomas in their book The Pragmatic Programmer."

Repetitive programming code leads to monotonous patterns. The programmer limits herself from using the tools she has available to her. The end result will be a limited web page. Therefore, if the programmer challenges her skillset, then the web page itself will illustrate the programmer's code skill level.

With regards to the benefits of not repeating code; repetitive code tends to become large. And if there is a problem in the first code and its template has been repeated numerous times, the errors will be repeated also. If there is less code then there is less chance for problems. If the future brings changes to the code then the less code there is to update the better off is the programmer.

The Definition of a Variable

According to Wikipedia, a variable in programming is "In computer programming, a variable or scalar is a storage location paired with an associated symbolic name (an identifier), which contains some known or unknown quantity or information referred to as a value. The variable name is the usual way to reference the stored value; this separation of name and content allows the name to be used independently of the exact information it represents. The identifier in computer source code can be bound to a value during run time, and the value of the variable may thus change during the course of program execution."

Object- Oriented Programming and Abstraction

According to Wikipedia, Object-Oriented Programming, also known as OOP "...is a programming paradigm based on the concept of 'objects', which are data structures that contain data, in the form of fields, often known as attributes; and code, in the form of procedures, often known as methods. A distinguishing feature of objects is that an object's procedures can access and often modify the data fields of the object with which they are associated (objects have a notion of 'this'). In object-oriented programming, computer programs are designed by making them out of objects that interact with one another.[1][2] There is significant diversity in object-oriented programming, but most popular languages are class-based, meaning that objects are instances of classes, which typically also determines their type."
